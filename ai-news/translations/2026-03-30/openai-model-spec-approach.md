# 我们对 Model Spec 的方法：背后的故事

> **发布方**：OpenAI  
> **发布日期**：2026 年 3 月 25 日  
> **译者**：Alice Larry

---

在 OpenAI，我们相信 AI 应该公平、安全，并能让更多人免费获取，以便他们用它来解决难题、创造机遇，并在健康、科学、教育、工作和日常生活等领域受益。我们认为，推动 AI 民主化是最佳路径：不是将 AI 的收益或控制权集中在少数人手中，而是让更多人能够使用、理解并参与塑造 AI。

这也是 OpenAI Model Spec 存在的核心原因。[Model Spec](https://model-spec.openai.com/) 是我们关于模型行为的正式框架。它定义了我们希望模型如何遵循指令、解决冲突、尊重用户自由，以及在用户每天提出的各种各样的问题中安全行事。更广泛地说，它是我们让预期模型行为变得明确的尝试——不仅体现在我们的训练过程中，而且以用户、开发者、研究人员、政策制定者和公众都能真正阅读、审查和讨论的形式呈现。

Model Spec 并非在声称我们的模型今天已经完美地按照这种方式运行。它在很大程度上是描述性的，但也是我们希望模型行为达到的目标。我们用它来明确预期行为，从而训练模型朝这一目标努力、针对它进行评估，并随着时间推移不断改进。

本文分享了 Model Spec 本身未包含的背景故事，包括其背后的哲学和机制：它的结构是什么、我们为何做出这些结构性选择，以及我们如何编写、实施和演进它。

## Model Spec 在 OpenAI 整体安全框架中的位置

Model Spec 是 OpenAI 更广泛的安全可问责 AI 方法的一部分。[Preparedness Framework](https://openai.com/index/updating-our-preparedness-framework/)（前沿准备框架）侧重于前沿能力带来的风险以及这些风险上升时所需的保障措施，而 Model Spec 则回答了一个不同但互补的问题：我们的模型在广泛情境下应该如何行事。进一步拉大视野，AI 韧性旨在解决一个更宏观的社会挑战——帮助社会在捕获先进 AI 收益的同时，减少越来越强大的系统部署过程中的动荡和新兴风险。总体而言，这些举措旨在帮助 AGI 转型过程变得渐进、迭代且民主可见：给人和机构时间去适应，同时建立将强大 AI 与人类利益保持一致所需的保障机制、问责机制和公众理解。

关于模型行为的公开透明，对公平性和安全性都至关重要。对公平性而言，人们需要理解 AI 为何以及如何对待他们——并能够在公平性问题出现时识别、质疑和解决它们。对安全性而言，随着 AI 系统能力的增强，人们和机构需要对其预期行为、所体现的权衡取舍以及如何随时间改进这些选择有更清晰的预期。这种可见性也支持韧性建设，让更多人有具体的东西可以审查、质疑和改进。

自 2024 年第一版发布以来，Model Spec 随着我们对用户偏好和需求的深入了解、覆盖范围的扩展以及对更强大能力的适应，以及从公众对模型行为和 Model Spec 的反馈中学习，已经发生了实质性演变。秉持[迭代部署](https://openai.com/safety/how-we-think-about-safety-alignment/)的精神，Model Spec 是一份不断演进的文档，涵盖背景价值观和明确的可见规则——并配有一个随着我们从现实世界部署和反馈中学习来修改各个元素的流程。我们也在投资于[集体对齐](https://openai.com/index/collective-alignment-aug-2025-updates/)等公众反馈机制，帮助人类保持对 AI 使用方式和 AI 行为塑造方式的控制。

对内，它为我们提供了预期行为的北极星，以及培训、评估和治理的共同框架。对外，它创造了一个公众参考点，供人们理解我们的方法、批评它，并随时间帮助改进它。

## Model Spec 的构成

Model Spec 由几种不同类型的模型指导组成。这是有意为之的。模型行为的不同部分需要以不同的方式处理，而一份有用的公开文档必须做的不仅仅是列举规则。

### 前言：高层意图

Model Spec 以高层意图开篇：对我们在系统层面试图优化什么以及原因的清晰阐述。

这段前言阐明了我们在追求使命过程中的三个目标：

- 迭代部署赋能开发者和用户的模型
- 防止我们的模型对用户或他人造成严重伤害
- 维持 OpenAI 的运营许可

然后它解释了我们如何在实践中平衡这些目标，将权衡取舍具体化，以支持后续更详细的原则。

重要的是，这段前言并非旨在作为对模型的直接指令。造福人类是 OpenAI 的目标，而不是我们希望模型自主追求的目标。相反，我们希望模型遵循一个指令链，其中包括 Model Spec 以及来自 OpenAI、开发者和用户的适用指令——即使某些人在特定情况下可能不同意其结果。我们认为这是正确的平衡，因为我们重视人类自主性和思想自由。如果我们训练模型根据我们自己对社会有益的看法来决定遵循哪些指令，OpenAI 就会处于在非常广泛的层面上裁决道德的位置。话虽如此，前言仍然很重要。当如何应用 Model Spec 存在歧义时，前言应有助于解决这个问题。

### 公开承诺

Model Spec 还包含公开承诺，这些承诺超越了直接可衡量的模型行为，涉及训练意图和部署约束。例如，我们的[红线原则](https://model-spec.openai.com/2025-12-18.html#red_line_principles)包含一项承诺：在 ChatGPT 等第一方部署中，我们绝不会使用系统消息来故意损害[客观性](https://model-spec.openai.com/2025-12-18.html#assume_objective_pov)或相关原则；[无其他目标](https://model-spec.openai.com/2025-12-18.html#no_other_objectives)则就我们优化模型响应以服务于用户利益而非收入或非有益的用户停留时长方面做出了承诺。

### 指令链：核心框架

Model Spec 的核心是指令链（Chain of Command）：一个用于决定在特定情况下应适用哪些指令的框架。它还涵盖了模型应如何处理未充分指定的指令，特别是在期望模型自主填补细节同时谨慎控制现实世界副作用的 agentic 场景中。

决定应适用哪些指令的基本思想很简单。指令可以来自不同来源，包括 OpenAI、开发者和用户。这些指令可能相互冲突。指令链解释了模型应如何解决这些冲突。

每个 Model Spec 策略和每条指令都被赋予一个[权限级别](https://model-spec.openai.com/2025-12-18.html#levels_of_authority)。模型被要求在冲突出现时优先遵循更高权限指令的字面意思和精神实质。如果用户要求帮助制造炸弹，模型应优先遵循[安全边界](https://model-spec.openai.com/2025-12-18.html#avoid_info_hazards)。如果用户要求被讽刺调侃，模型通常应优先考虑该请求，而不是 Model Spec 中优先级较低的[反滥用政策](https://model-spec.openai.com/2025-12-18.html#avoid_abuse)。

这种结构让我们能够定义一小组不可覆盖的规则，以及更多默认行为规则。这就是我们在安全约束内尽量最大化用户自由和开发者控制权的方式。

**硬规则**是用户或开发者不可覆盖的明确边界（在 Model Spec 的术语中，这些是"根级"或"系统级"指令）。它们大多是禁止性的，要求模型避免可能导致灾难性风险或直接人身伤害、违法或破坏指令链的行为。我们预期 AI 将成为社会的基础性技术，类似于基本互联网基础设施，因此我们只在认为确有必要时才施加可能限制思想自由的规则，以覆盖将与其互动的广泛开发者和用户群体。在 Model Spec 中，[保持在边界内](https://model-spec.openai.com/#stay_in_bounds)包含了应对具体现实世界安全风险的硬规则，[未成年人原则](https://model-spec.openai.com/#chatgpt_u18)则为 18 岁以下用户叠加了额外保障。

**默认行为**是可覆盖的起点：当用户或开发者未指定偏好时，助手的"最佳猜测"行为。我们使用默认值让行为在规模上可预测和可控，这样人们无需每次都编写定制指令集就能预期会发生什么。默认值保留了可引导性：用户和开发者可以在安全边界内明确引导语气、深度、格式，甚至观点。指导方针级别的默认值（如语气或风格）被设计为可隐式引导，而用户级别的默认值（如真实性和客观性）则是信任和可预测性的锚点，只能通过明确指令覆盖。这些不应基于感觉悄悄漂移；如果用户想要不同的事实立场，将其作为明确指令可以保持这种转变的透明性和可见性。这些默认值体现在[共同追求真相](https://model-spec.openai.com/#seek_truth)、[尽力做到最好](https://model-spec.openai.com/#best_work)和[使用适当风格](https://model-spec.openai.com/#style)中，包括关于诚实和客观性的规范、避免谄媚，以及直接性和情境适当的温暖感与专业性等互动规范。

### 解释辅助工具

除了层级结构本身，Model Spec 还使用解释辅助工具来帮助模型（和人类）在灰色地带一致地应用它。这些工具包括：

- **决策标准**：帮助模型在灰色地带做出一致选择，而不假装存在单一的机械规则。例如，Model Spec 关于[控制副作用](https://model-spec.openai.com/2025-12-18.html#control_side_effects)的指导列出了一些考量因素：最小化不可逆行动、保持行动与目标成比例、减少不良意外、倾向可逆方法——这些应与其他目标（如快速有效地完成任务）相平衡。
- **具体示例**：展示原则在实践中应如何应用。这些是简短的提示与响应示例，通常包括合规和不合规两种响应，通常针对重要决策边界附近的困难提示。目标不是模拟一段完整的真实对话，而是清晰地呈现关键区别，并以同时展示所需响应风格的方式呈现。

我们保持示例数量相对较少，专注于最具信息价值的示例。更广泛的评估套件则有助于覆盖更长尾的情况。

## 关于 Model Spec 的几点说明

**Spec 是接口，而非实现**。它描述我们想要的行为，而非每个产生该行为的细节。我们尽量避免将其锚定到实现细节，例如内部 token 格式或特定行为的确切训练方案，因为即使预期行为不变，这些细节也可能发生变化。Model Spec 的主要受众不是模型，而是人类：它旨在帮助 OpenAI 员工、用户、开发者、研究人员和政策制定者理解、讨论并决定预期行为。

**Spec 描述的是模型，而非整个产品**。它由我们的[使用政策](https://openai.com/policies/usage-policies)补充，后者概述了我们对人们使用 API 和 ChatGPT 方式的期望。用户与之交互的系统不仅仅包含模型本身：自定义指令和记忆等产品功能、监控、政策执行以及其他层面也同样重要。安全远不止于模型行为，我们相信[纵深防御](https://openai.com/safety/how-we-think-about-safety-alignment/)。

**Spec 也不是我们整个训练堆栈或每项内部政策区别的完整记录**。目标不是捕获每一个细节，而是以与我们预期模型行为完全一致的方式，让最重要的行为决策变得可理解。

## 为什么要在 Spec 中写这么多内容

将如此多内容放入 Spec 而不是假设读者——或模型——能从几个高层目标推断出一切，有几个原因。

首先，**Model Spec 是一个透明度和问责工具**。它旨在鼓励有意义的公众反馈。一个清晰的公开目标帮助人们判断某个行为是缺陷还是特性。它为批评和具体反馈提供了一个稳定的参考点。这就是为什么我们[开源](https://github.com/openai/model_spec)了 Model Spec，并选择在公众视野中迭代。自第一版发布以来，许多变化都是基于公众反馈做出的，这些反馈通过各种机制收集，包括反馈表、公众批评以及收集民主意见的刻意[努力](https://openai.com/index/collective-alignment-aug-2025-updates/)。

其次，**Model Spec 是 OpenAI 内部的一个协调工具**。它为研究、产品、安全、政策、法律、传播和其他职能部门的人员提供了一个讨论模型行为的共同词汇，以及提出和审查变更的机制。
