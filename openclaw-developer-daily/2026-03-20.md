### 1. Context Engine 支持 Transcript 维护钩子 [#51191]

**PR**: [feat: add context engine transcript maintenance](https://github.com/openclaw/openclaw/pull/51191)  
**作者**: @jalehman | **规模**: XL (+1305/-107)

**背景**：Context Engine 插件（如 lossless-claw）通过 DAG 压缩保持上下文窗口有界，但底层 JSONL transcript 文件会无限增长。已被摘要的大型工具结果仍然占据磁盘空间，导致生产环境出现数百 MB 的 session 文件。

**核心改动**：

**A. 新增 `maintain()` 生命周期钩子**：
```typescript
// context-engine/types.ts
interface ContextEngine {
  // 新增可选方法
  maintain?(ctx: ContextEngineRuntimeContext): Promise<ContextEngineMaintenanceResult>;
}

// 调用时机：bootstrap 后、成功 turn 后、compaction 后
```

**B. Transcript 重写 API**：
```typescript
// 插件请求重写特定条目
interface TranscriptRewriteRequest {
  entryId: string;
  replacement: TranscriptEntry | null;  // null = 删除
}

// Runtime 负责执行重写
await runtimeContext.rewriteTranscriptEntries([
  { entryId: "msg-123", replacement: { ...truncatedToolResult } },
  { entryId: "msg-456", replacement: null }  // 删除
]);
```

**C. 重构通用重写逻辑**：
```typescript
// 从 tool-result-truncation.ts 抽取通用逻辑
// transcript-rewrite.ts — 202 行通用 branch-and-reappend 重写器

async function rewriteTranscript(
  sessionPath: string,
  rewrites: TranscriptRewriteRequest[],
  options: { preserveLabels?: boolean; acquireLock?: boolean }
): Promise<TranscriptRewriteResult> {
  // 1. 获取 session 写锁
  await acquireSessionWriteLock(sessionPath);
  
  // 2. 读取现有 transcript
  const entries = await parseJSONL(sessionPath);
  
  // 3. 应用重写，保留非消息类型条目
  const rewritten = applyRewrites(entries, rewrites);
  
  // 4. 重新映射 compaction keep markers
  remapCompactionMarkers(rewritten);
  
  // 5. 原子写入（temp + rename）
  await atomicWriteJSONL(sessionPath, rewritten);
}
```

**影响**：Context Engine 插件现在可以安全地请求清理已摘要的内容，解决 session 文件无限增长问题。

function wrapStreamFnSanitizeMalformedToolCalls(
  streamFn: StreamFn,
  transcriptPolicy: TranscriptPolicy
): StreamFn {
  return async (context) => {
    // 1. 清理 malformed tool-call blocks
    const sanitized = context.messages.map(msg => {
      if (msg.role === "assistant" && msg.toolCalls) {
        msg.toolCalls = msg.toolCalls.filter(tc => 
          tc.id && tc.name && tc.arguments
        );
      }
      return msg;

    // 2. 修复孤立的 toolResult
    const repaired = sanitizeToolUseResultPairing(sanitized);

### 3. Matrix 重启后消息重复 [#50922]

**PR**: [Matrix: dedupe replayed inbound events on restart](https://github.com/openclaw/openclaw/pull/50922)  
**作者**: @gumadeiras | **规模**: XL (+1087/-18)

**问题**：Gateway 重启后，Matrix 会重放启动前的历史事件，导致消息重复处理。

**修复**：
```typescript
// inbound-dedupe.ts — 持久化去重
class InboundDedupe {
  private seen: Map<string, number>;  // eventId -> timestamp
  private persistPath: string;
  
  async markSeen(roomId: string, eventId: string): Promise<boolean> {
    const key = `${roomId}:${eventId}`;
    if (this.seen.has(key)) {
      return false;  // 已处理
    }
    
    this.seen.set(key, Date.now());
    await this.persist();  // 持久化到磁盘
    return true;
  }
}

// handler.ts
async function handleEvent(event: RoomEvent) {
  const isNew = await dedupe.markSeen(event.roomId, event.eventId);
  if (!isNew) {
    log.debug("Skipping replayed event", { eventId: event.eventId });
    return;  // 跳过重复
  }
  
  // 正常处理...
}
```

**关键改进**：
- 仅在成功处理或有意忽略后才标记为已处理
- 瞬态失败可以重试
- 关闭时先停止 sync、drain 解密队列、再持久化
