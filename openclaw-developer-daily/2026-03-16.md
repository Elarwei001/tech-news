### 1. Claude 插件市场集成：一行命令安装 Claude 生态插件 [#48058]

**PR**: [Plugins: add Claude marketplace registry installs](https://github.com/openclaw/openclaw/pull/48058)  
**作者**: @vincentkoc | **规模**: XL (+1643/-144)

**背景**：Claude 用户习惯使用 `plugin@marketplace` 格式安装插件，但 OpenClaw 此前只支持 npm 包和本地路径安装。Claude 插件市场的大量优质插件无法直接被 OpenClaw 使用。

**新功能**：

1. **市场注册表解析**：读取 `~/.claude/plugins/known_marketplaces.json` 本地注册表
2. **简写安装**：`openclaw plugins install superpowers@claude-plugins-official`
3. **市场列表**：`openclaw plugins marketplace list <marketplace>`
4. **更新支持**：安装元数据持久化，支持 `openclaw plugins update`

**核心实现**：
```typescript
// src/plugins/marketplace.ts
export async function resolveMarketplacePlugin(
  shorthand: string // 如 "superpowers@claude-plugins-official"
): Promise<ResolvedMarketplacePlugin> {
  const [pluginName, marketplaceName] = shorthand.split('@');
  
  // 读取 Claude 本地注册表
  const registry = await loadClaudeMarketplaceRegistry();
  const marketplace = registry.get(marketplaceName);
  if (!marketplace) {
    throw new Error(`Unknown marketplace: ${marketplaceName}`);
  }
  
  // 从市场源解析插件
  const pluginEntry = await marketplace.resolvePlugin(pluginName);
  return {
    name: pluginName,
    source: pluginEntry.source, // git/path
    marketplace: marketplaceName,
  };
}
```

**安装元数据持久化**：
```typescript
// 记录安装来源，支持后续更新
{
  "plugins": {
    "superpowers": {
      "source": "marketplace",
      "marketplace": "claude-plugins-official",
      "installedAt": "2026-03-16T08:46:07Z"
    }
  }
}
```

**影响**：Claude 插件生态无缝接入 OpenClaw，`superpowers`、`semgrep` 等热门插件一行命令安装。

### 2. 跨 Channel 统一交互组件：一套代码，三平台渲染 [#47715]

**PR**: [Channels/Plugins: centralize shared interactive payloads](https://github.com/openclaw/openclaw/pull/47715)  
**作者**: @vincentkoc | **规模**: XL (+1951/-559)

**背景**：此前 Telegram 按钮用 `buttons`，Discord 用 `components`，Slack 用 `blocks`——三套完全不同的 API。插件开发者要么为每个 channel 写三遍代码，要么放弃交互功能。

**新方案**：引入 `interactive` 统一模型：

```typescript
// 统一交互 payload
interface InteractivePayload {
  buttons?: Array<{
    label: string;
    action: string;  // callback action
    style?: "primary" | "secondary" | "danger";
  }>;
  select?: {
    placeholder?: string;
    options: Array<{ label: string; value: string }>;
    action: string;
  };
}

// 使用示例
reply({
  text: "选择操作：",
  interactive: {
    buttons: [
      { label: "确认", action: "confirm", style: "primary" },
      { label: "取消", action: "cancel", style: "secondary" },
    ]
  }
});
```

**Channel 适配器自动转换**：
```typescript
// extensions/telegram/src/interactive.ts
export function renderInteractivePayload(
  payload: InteractivePayload
): TelegramInlineKeyboard {
  return {
    inline_keyboard: payload.buttons.map(btn => [{
      text: btn.label,
      callback_data: btn.action,
    }])
  };
}

// extensions/slack/src/interactive.ts  
export function renderInteractivePayload(
  payload: InteractivePayload
): SlackBlocks {
  return {
    blocks: [{
      type: "actions",
      elements: payload.buttons.map(btn => ({
        type: "button",
        text: { type: "plain_text", text: btn.label },
        action_id: btn.action,
        style: btn.style === "danger" ? "danger" : "primary",
      }))
    }]
  };
}
```

**兼容性**：原有的 channel-specific payload（`buttons`、`components`、`blocks`）仍然有效，作为 escape hatch。

**影响**：插件开发者只需写一套交互代码，自动适配 Telegram/Discord/Slack/Feishu 等所有支持的 channel。

### 3. 浏览器支持非 Chrome 浏览器：Brave/Edge 一键接入 [#48170]

**PR**: [Browser: support non-Chrome existing-session profiles via userDataDir](https://github.com/openclaw/openclaw/pull/48170)  
**作者**: @velvet-shark | **规模**: L (+650/-126)

**背景**：`existing-session` 浏览器配置原本只支持 Chrome，因为它硬编码了 Chrome 的用户数据目录。使用 Brave、Edge 等 Chromium 系浏览器的用户无法享受 `profile="user"` 带来的登录态复用。

**新功能**：`userDataDir` 配置项指向任意 Chromium 浏览器：

```yaml
# ~/.openclaw/openclaw.yaml
browser:
  profiles:
    brave-live:
      driver: existing-session
      attachOnly: true
      userDataDir: "~/Library/Application Support/BraveSoftware/Brave-Browser"
    
    edge-live:
      driver: existing-session
      attachOnly: true
      userDataDir: "~/Library/Application Support/Microsoft Edge"
```

**核心实现**：
```typescript
// src/browser/existing-session.ts
export async function resolveDevToolsPort(
  profile: ExistingSessionProfile
): Promise<number> {
  const userDataDir = profile.userDataDir ?? getDefaultChromeUserDataDir();
  
  // 读取 DevToolsActivePort 文件
  const portFile = path.join(userDataDir, "DevToolsActivePort");
  if (!fs.existsSync(portFile)) {
    throw new Error(
      `DevToolsActivePort not found at ${portFile}. ` +
      `Ensure the browser is running with --remote-debugging-port.`
    );
  }
  
  const content = fs.readFileSync(portFile, "utf-8");
  return parseInt(content.split("\n")[0], 10);
}
```

**Doctor 自动迁移**：
```bash
$ openclaw doctor --fix
# 检测到 driver: "extension" (旧格式)
# 自动迁移为 driver: "existing-session"
```

**影响**：Brave/Edge/Vivaldi 等 Chromium 浏览器用户可以复用登录态，无需切换到 Chrome。

### 1. Gateway 重启后 Subagent 任务丢失 [#47719]

**PR**: [fix: resume orphaned subagent sessions after SIGUSR1 reload](https://github.com/openclaw/openclaw/pull/47719)  
**作者**: @joeykrug | **规模**: L (+959/-10)

**问题**：Gateway 热重载（SIGUSR1）期间的 subagent 任务会"丢失"：
1. `openclaw gateway restart` CLI 直接发送 SIGUSR1，不检查活跃任务
2. 重启后恢复的任务可能卡住——等待路径在 gateway 就绪前被重新设置
3. 中断的 turn 不会自动恢复

**修复方案**：

**A. 孤儿恢复模块**：
```typescript
// src/agents/subagent-orphan-recovery.ts
export async function scanAndResumeOrphans(): Promise<void> {
  await delay(5000); // 等待 gateway bootstrap
  
  const orphans = await findOrphanedRuns();
  for (const orphan of orphans) {
    if (!orphan.abortedLastRun) continue;
    
    // 注入恢复消息触发新 LLM turn
    const newRunId = await callGateway({
      method: "agent",
      sessionKey: orphan.sessionKey,
      message: "[系统] 检测到中断，正在恢复..."
    });
    
    // 重新映射 runId 以正确跟踪完成事件
    registry.remapRun(orphan.runId, newRunId);
  }
}
```

**B. 可配置延迟超时**：
```yaml
gateway:
  reload:
    deferralTimeoutMs: 300000  # 5 分钟（原 90s）
```

**影响**：长时间运行的 subagent 任务在 gateway 重启后能自动恢复，不再丢失。
