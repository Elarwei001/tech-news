### 3. 发送持久化 + 过期 Cron 保护 [#50092]

**PR**: [Persist outbound sends and skip stale cron deliveries](https://github.com/openclaw/openclaw/pull/50092)  
**作者**: @tyler6204 | **规模**: XL (+1381/-33)

**背景**：两个独立但相关的问题：(1) agent 主动发送消息后 follow-up 丢失目标上下文；(2) cron 任务延迟执行后仍然发送过期内容（比如早报延迟 10 小时后仍然发出）。

**核心改动**：

**A. Outbound Session 持久化**：
```typescript
// 发送消息后，目标路由状态持久化到 session transcript
await sessionTranscript.append({
  role: "assistant",
  content: message,
  metadata: {
    outboundTarget: { channel: "slack", target: "#general" },
    deliveredAt: Date.now()
  }
});

// follow-up send 自动恢复目标上下文
const lastOutbound = transcript.findLast(isOutboundMessage);
if (lastOutbound?.metadata?.outboundTarget) {
  return lastOutbound.metadata.outboundTarget;  // 不需要重新指定
}
```

**B. BlueBubbles 自动创建 Chat**：
```typescript
// 发送给新号码时自动创建 chat
async function sendMessage(phoneNumber: string, message: string) {
  let chat = await findChat(phoneNumber);
  if (!chat) {
    chat = await blueBubbles.createChat(phoneNumber);  // 新增
  }
  await chat.send(message);
}
```

**C. Stale Cron Delivery Guard**：
```typescript
// 超过 3 小时的 cron 结果视为 stale
const STALE_THRESHOLD_MS = 3 * 60 * 60 * 1000;

function shouldDeliverCronResult(run: CronRun): boolean {
  const scheduledAt = run.scheduledAt;
  const now = Date.now();
  
  if (now - scheduledAt > STALE_THRESHOLD_MS) {
    log.warn(`Skipping stale cron delivery: scheduled ${scheduledAt}, now ${now}`);
    // 标记为 handled，防止 fallback 重播
    run.deliveryStatus = "stale-skipped";
    return false;
  }
  return true;
}
```

**场景示例**：
```
# 早上 8:00 scheduled 的 cron
# 因 gateway 故障延迟到下午 6:00 才执行完成
# 改前：发送 "今日天气：早间有雾" — 完全过期
# 改后：跳过 delivery，不发送过期内容
```

### 1. Plugin 模块分裂导致命令消失 [#50418]

**PR**: [fix(plugins): share split-load singleton state](https://github.com/openclaw/openclaw/pull/50418)  
**作者**: @huntharo | **规模**: S (+102/-19)

**问题**：Rolldown 代码分割会将同一模块复制到多个 chunk，导致 `listeners` Map 和 `pluginCommandRegistry` 出现多个副本。一个 chunk 注册命令，另一个 chunk 读取，结果 `/voice`、`/phone`、`/pair` 等插件命令在 Telegram/Discord 消失。

**修复**：
```typescript
// 改前 — 模块局部状态
const listeners = new Map<string, Listener>();

// 改后 — globalThis singleton
const LISTENERS_KEY = Symbol.for("openclaw.whatsapp.listeners");
const listeners = (globalThis as any)[LISTENERS_KEY] ??= new Map();

// pluginCommandRegistry 同理
const REGISTRY_KEY = Symbol.for("openclaw.plugins.commands");
const registry = (globalThis as any)[REGISTRY_KEY] ??= new Map();
```

**影响**：修复 `gateway:watch` 开发模式下插件命令丢失问题。

### 2. Volcano Engine DeepSeek 返回 400 错误 [#49877]

**PR**: [fix: strip prompt_cache_key for non-OpenAI openai-responses endpoints](https://github.com/openclaw/openclaw/pull/49877)  
**作者**: @ShaunTsai | **规模**: S (+100/-1)

**问题**：OpenClaw 向 OpenAI Responses API 注入 `prompt_cache_key` 和 `prompt_cache_retention` 字段以支持 prompt caching，但 Volcano Engine 等非 OpenAI 兼容端点不识别这些字段，返回 `HTTP 400: unknown field "prompt_cache_key"`。

**修复**：
```typescript
// openai-stream-wrappers.ts
function createOpenAIResponsesContextManagementWrapper(baseUrl: string) {
  const stripPromptCache = !isDirectOpenAIBaseUrl(baseUrl);
  
  return (payload: OpenAIResponsesPayload) => {
    if (stripPromptCache) {
      delete payload.prompt_cache_key;
      delete payload.prompt_cache_retention;
    }
    return payload;
  };
}

// isDirectOpenAIBaseUrl 识别：
// - api.openai.com
// - chatgpt.com
// - *.openai.azure.com
// 其他所有 baseUrl 都 strip cache 字段
```
