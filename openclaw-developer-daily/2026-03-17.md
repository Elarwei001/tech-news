### 1. 内嵌 Pi 运行 Bundle MCP 工具：Claude 插件生态完整接入 [#48611]

**PR**: [Agents: run bundle MCP tools in embedded Pi](https://github.com/openclaw/openclaw/pull/48611)  
**作者**: @vincentkoc | **规模**: XL (+2051/-30)

**背景**：Claude 市场上大量插件（如 Context7）通过 MCP 协议提供工具。此前 OpenClaw 可以导入 bundle 的 MCP 配置，但内嵌 Pi agent 实际上无法执行这些工具——配置导入了，功能却不工作。

**新功能**：Bundle MCP 运行时桥接器：

1. **自动发现**：读取 bundle 的 `.mcp.json`，启动对应的 stdio MCP server
2. **工具暴露**：将 MCP server 的工具列表注入 Pi 的 custom tools
3. **统一执行**：正常 Pi turn 和 compaction session 共享同一运行时

**核心实现**：
```typescript
// src/agents/pi-bundle-mcp-tools.ts
export async function createBundleMcpToolRuntime(
  enabledBundles: BundleEntry[]
): Promise<BundleMcpToolRuntime> {
  const runtime = new BundleMcpToolRuntime();
  
  for (const bundle of enabledBundles) {
    const mcpConfig = await loadBundleMcpConfig(bundle.root);
    if (!mcpConfig?.servers) continue;
    
    for (const [name, server] of Object.entries(mcpConfig.servers)) {
      // 启动 stdio MCP server
      const client = await launchMcpStdioServer({
        command: server.command,
        args: server.args,
        cwd: bundle.root, // 关键：相对路径基于 bundle 根目录
      });
      
      // 获取并注册工具
      const tools = await client.listTools();
      for (const tool of tools) {
        // 跳过重名工具，避免 shadowing
        if (runtime.hasToolNamed(tool.name)) {
          log.warn(`Duplicate tool name "${tool.name}", skipping`);
          continue;
        }
        runtime.registerTool(tool, client);
      }
    }
  }
  
  return runtime;
}
```

**使用示例**：
```yaml
# 启用 Context7 bundle 后，Pi agent 自动获得其 MCP 工具
plugins:
  entries:
    context7:
      enabled: true
      source: marketplace
      marketplace: claude-plugins-official
```

**影响**：Claude 插件市场的 MCP 工具现在真正可用，而非只是配置占位符。

### 1. 安全：封堵 JVM/Python/.NET 环境变量注入 [#49025]

**PR**: [fix(security): block JVM, Python, and .NET env injection vectors in host exec sandbox](https://github.com/openclaw/openclaw/pull/49025)  
**作者**: @ademczuk | **规模**: XS (+23/-2)

**问题**：攻击者可通过环境变量注入恶意代码：

| 变量 | 攻击向量 | 运行时 |
|------|----------|--------|
| `JAVA_TOOL_OPTIONS` | 注入 `-javaagent:/path/evil.jar` | JVM |
| `_JAVA_OPTIONS` | 同上（HotSpot/OpenJDK 非标准） | JVM |
| `JDK_JAVA_OPTIONS` | Java 9+ 标准化替代 | JVM |
| `PYTHONBREAKPOINT` | 重定向 `breakpoint()` 到 `os.system` | Python 3.7+ |
| `DOTNET_STARTUP_HOOKS` | 在 `Main()` 前加载任意程序集 | .NET Core 3.0+ |

**修复**：将这 5 个变量加入 `blockedKeys`：
```json
// src/infra/host-env-security-policy.json
{
  "blockedKeys": [
    // 已有的 LD_*, DYLD_*, PYTHONPATH 等
    "JAVA_TOOL_OPTIONS",
    "_JAVA_OPTIONS", 
    "JDK_JAVA_OPTIONS",
    "PYTHONBREAKPOINT",
    "DOTNET_STARTUP_HOOKS"
  ]
}
```

**影响**：跨语言运行时注入攻击被封堵，exec sandbox 更安全。
