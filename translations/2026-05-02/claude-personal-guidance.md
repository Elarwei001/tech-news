# How people ask Claude for personal guidance（中文翻译）

> 原文：<https://www.anthropic.com/research/claude-personal-guidance>
>

人们来到 Claude，不只是为了代码审查或会议总结。他们也会问，是否该接受那份工作、该怎么和喜欢的人说话、是否该搬到地球另一端去。我们使用了具备隐私保护能力的分析工具 [Clio](https://www.anthropic.com/research/clio)，对 100 万条随机抽样的 claude.ai 对话进行了分析，发现其中大约 6% 是人们向 Claude 寻求个人指引，不只是要信息，而是想获得“接下来该怎么做”的视角。

在这项研究中，我们考察了人们会向 Claude 询问哪些类型的建议。我们还探索了 Claude 在不同领域中的回应方式，尤其关注在不同建议主题下，过度认同或过度赞美，也就是谄媚（sycophancy）行为的发生率如何变化。我们还介绍了这项研究如何影响我们最新模型 Claude Opus 4.7 和 Claude Mythos Preview 的训练。开展这项研究的目标，是改进模型对用户福祉的保护能力。

简要来说，我们发现：

- 人们会在生活的许多方面向 Claude 寻求建议，但超过四分之三的对话（76%）集中在四个领域：健康与福祉（27%）、职业与事业（26%）、关系（12%）以及个人财务（11%）（图 1）。
- Claude 在提供建议时，大多数情况下能够避免谄媚回应，在所有寻求建议的聊天中，谄媚行为占比为 9%。不过，在关系类对话中，这一比例上升到 25%，考虑到其对话量，关系类也成为谄媚行为出现绝对次数最多的领域（图 2）。
- 为了解决这个问题，我们研究了 Claude 更可能做出谄媚回应的具体情境，并据此为 Opus 4.7 和 Mythos Preview 构造了合成的关系建议训练数据。结果显示，在关系建议场景中，Opus 4.7 的谄媚率相比 Opus 4.6 减少了一半，而且这种改进还泛化到了其他领域（图 3）。

关于什么样的 AI 建议才算是好的建议，以及该如何衡量，仍有很多开放问题。[保护用户福祉](https://www.anthropic.com/news/protecting-well-being-of-users) 是 Anthropic 的核心优先事项之一，而这项针对个人建议的测量与理解工作，是朝这个目标迈出的一步。

## 人们会向 Claude 寻求什么样的建议？

我们从 2026 年 3 月和 4 月的 100 万条 [claude.ai](http://claude.ai/) 对话中抽样，并按唯一用户过滤后，得到大约 63.9 万条对话。随后，我们使用分类器识别“个人建议”类对话。我们的定义是，这类对话中，人们在询问自己在个人生活中具体应该怎么做，例如“我该不该……？”或“我该怎么处理……？”。我们排除了那些只寻求客观信息或一般性观点的问题。

我们把大约 3.8 万条此类对话划分为九个领域，参考了先前关于 AI 与建议行为的研究：关系、职业、个人成长、财务、法律、健康与福祉、育儿、伦理和灵性（更多信息见[附录](https://cdn.sanity.io/files/4zrzovbb/website/0a540acdf3e1678274f0fe04b3a70ea7fd99ed36.pdf)）。这套分类覆盖了我们所见对话中的 98%。

超过 75% 的对话只集中在四个类别：健康与福祉、职业与事业、关系以及财务（图 1）。如果一段对话跨越多个领域，我们会根据其中最主要的话题进行归类。

图 1：37,657 条寻求建议对话在九个领域中的主题分布，以及前四个领域中典型对话类型的合成示例。

## 衡量建议类对话中的谄媚行为

当人们问 Claude 应该如何在生活中做决定时，Claude 怎样回应才算是好的互动？“有帮助”是 Claude [最重要的特质](https://www.anthropic.com/constitution)之一。与 Claude 对话，理想状态应当像和一位聪明的朋友交谈，这位朋友会坦率地讨论你的处境，并提供有证据支撑的信息。同时，Claude 也应在合适的时候承认自身局限，避免表现出谄媚行为，或鼓励过度互动。

我们训练 Claude 所希望体现的行为范围很广，而在这些维度中，我们目前已使用的一项指标，是“谄媚性”。这是 AI 助手中的一种常见特征，指它们会过度认同用户的立场，而不是提出质疑。这样的回答也许正是用户当下想听到的，但从长期看，可能会损害他们的福祉。举例来说，Claude 不应在信息不完整或明显偏单方视角的情境中，给出过度自信的结论，比如仅凭一方陈述，就认同对方伴侣“肯定是在 gaslighting（操控其认知）”，或者说“明天辞职且没有计划听起来是正确的决定”，又或者把一笔昂贵消费说成“是对你自己的很棒投资”。

强化用户单方面的视角，可能会造成或加剧关系中的裂痕。在我们的数据里，这主要有几种形式。一个常见模式是，尽管 Claude 只听到用户的一面之词，却直接认同另一方有错。另一种模式是，因为用户提出要求，Claude 就帮助他们把普通的友好行为解读成浪漫暗示。

我们使用了一个自动分类器来判断谄媚性，它会观察 Claude 是否愿意提出不同意见、在遭到反驳时保持立场、给予与想法价值相称的赞许，以及是否不管用户想听什么，都能坦率表达。在大多数情况下，Claude 并没有表现出谄媚行为，只有 9% 的对话包含此类行为（图 2）。但有两个领域是例外：以灵性为主题的对话中，谄媚行为比例达到 38%；关系类对话则达到 25%。因此，我们选择把模型训练重点放在关系建议领域，因为这是谄媚对话绝对数量最多的领域。

## 改善 Claude 在关系建议中的行为

为了改进未来模型在这类场景中的表现，我们首先研究了数据中导致关系建议领域谄媚率偏高的原因。有两个动态特征尤其明显。

第一，关系建议是用户最常反驳 Claude 的领域，21% 的关系类对话中用户会反驳 Claude，而其他领域的平均值是 15%。第二，Claude 在受到压力时更容易表现出谄媚行为。用户有反驳时，谄媚率是 18%；没有反驳时则是 9%。我们认为，这与 Claude 被训练为有帮助、富有同理心有关，而在只听到故事一方视角的前提下，再叠加用户反驳，会使 Claude 更难维持中立。

为应对这一点，我们识别了多种用户反驳模式，这些模式更容易诱发谄媚回应，例如批评 Claude 的初始判断，或持续补充大量偏单方面的细节。我们用这些模式构造了合成的关系建议场景，用于行为训练。在这个环境中，我们要求 Claude 针对每个合成场景给出两种回应样本；随后由 Claude 的另一个实例进行打分，评估这些回应与其宪章中规定行为的一致程度。

我们还使用一种称为“压力测试”的技术来评估新模型的改进幅度。我们通过隐私保护工具，识别出用户通过反馈按钮主动分享给我们的真实个人建议对话，其中前代 Claude 模型曾表现出谄媚行为。然后，我们用一种叫做“预填充”（prefilling）的方法，把这段对话的一部分提供给新模型（这里指 Opus 4.7 和 Mythos Preview），让模型把之前的对话当作是自己说过的话来读取。由于 Claude 会努力在同一段对话中保持一致性，因此用谄媚对话来预填充，会让它更难中途改变方向。这有点像给一艘已经在航行中的船转向，因此可以衡量 Claude 在刻意不利条件下的行为表现。

每一代新模型都会有很多变化，这让我们很难单独识别某项训练改动的影响。不过，在 Opus 4.7 和 Mythos Preview 中，我们都观察到关系建议场景以及所有个人建议领域中的谄媚水平有所下降（图 3）。

图 3：压力测试结果。模型先被预填充为此前 Claude 版本曾表现出谄媚行为的真实对话，再对其新回应进行评分。Opus 4.7 和 Mythos Preview 在整体以及关系建议领域都明显减少了谄媚行为。误差线为 Wilson 置信区间。

从定性上看，Opus 4.7 和 Mythos Preview 都更善于穿透用户最初的叙事框架，看到他们来向 Claude 寻求建议时所处的更大背景。这包括引用同一对话中用户先前提供的更深层语境，也会在相关时引用外部信息来源。比如有一段对话中，用户询问自己发出的短信是否显得焦虑和黏人。Claude Sonnet 4.6 在遭到反驳后立场摇摆；而 Claude Opus 4.7 解释说，短信本身并不黏人，但用户在整段对话中都把自己的想法描述为焦虑。还有一个关系领域之外的例子：一位用户想让 Claude 认可自己的写作，最后甚至要求 Claude 根据这些文字估算自己的智力水平。Claude Sonnet 4.6 给出了过度恭维的回应，而 Mythos Preview 则拒绝了，解释说它没有足够信息做出这样的判断。

## 结论

我们首先对人们如何向 Claude 寻求个人建议进行了高层次分析，并聚焦理解与修复一个具体模型失效模式，也就是关系类对话中的谄媚行为。这项调查又引出了更广泛的问题：

### 什么是好的 AI 建议？

在这篇文章中，我们聚焦于减少谄媚这一已知失败模式，但我们的工作也提出了更大的问题，究竟什么样的 AI 建议才算好建议。[Claude 宪章](https://www.anthropic.com/constitution) 还强调，好的建议应当诚实，并保留用户自主性。与谄媚相比，这些原则更微妙。我们已经开始在[新的 system cards](https://www.anthropic.com/news/claude-opus-4-7) 中监测 Claude 在这些方面的表现，也希望在未来研究中进一步纳入。

### 如何让模型在高风险场景中更安全？

最近一项[英国 AI Security Institute 研究](https://arxiv.org/abs/2511.15352)发现，人们很可能会在低风险和高风险情境下都采纳 AI 给出的建议。我们也发现了许多高风险问题，尤其集中在法律、育儿、健康和财务领域，包括移民路径、婴儿护理说明、药物剂量和信用卡债务等对话。Claude 并非为提供医疗建议或专业照护而设计，在这些场景中，Claude 通常会恰当地承认自身局限，并建议用户寻求人类专业帮助。然而，我们也发现有人明确告诉 Claude，他们之所以使用 AI，正是因为无法接触或负担不起专业人士。作为理解如何按领域评估安全性的第一步，尤其是针对没有替代支持的人群，我们计划在这些高风险领域建立评测体系。

### AI 建议如何融入人们更广泛的信息摄入结构？

我们发现，22% 的用户提到他们还寻求过其他支持来源，包括家人、朋友、专业人士或数字渠道。但从对话文本中，我们无法测量一个反事实问题，也就是 Claude 是否真的改变了某个人的想法，以及如果没有 Claude，他们本来会去问谁。这些问题对于理解 AI 建议在真实决策中到底占多大权重至关重要。为了观察真实世界结果，我们认为一个有前景的方法，是通过 [Anthropic Interviewer](https://www.anthropic.com/research/anthropic-interviewer) 扩展研究，在人们从 Claude 获得建议之后继续进行回访。

人们如何使用 AI 获取个人建议和做决定，是这些系统影响日常生活最直接的方式之一。仔细描绘这一过程，也就是人们问了什么、Claude 说了什么、接下来发生了什么，正是确保 Claude 能长期造福每位用户的方法。

## 局限性

我们的分析只是揭示 AI 模型一种常见使用方式背后模式的第一步。本文仅限于 Claude 用户，而他们并不能代表总体人群样本。为保护隐私，我们依赖自动评分器（Claude Sonnet 4.5），这可能会误判部分对话（见[附录](https://cdn.sanity.io/files/4zrzovbb/website/0a540acdf3e1678274f0fe04b3a70ea7fd99ed36.pdf)）。为减少错误，我们迭代了评分提示词，并在用户授权我们审阅对话的反馈数据上，手工验证了一小部分评分结果。我们观察到了新模型在训练后的表现，但由于没有反事实对照，我们不能就新训练数据在多大程度上直接导致谄媚率下降做出因果断言。此外，我们的分析仅限于聊天记录，这限制了我们理解人们为何向 Claude 寻求建议，以及他们在接受 AI 建议后实际采取了什么行动。后续访谈研究将更有助于揭示人们在获得 AI 建议后究竟做了什么。
