# OpenAI 对 Axios 开发者工具供应链攻击事件的回应

*原文：[Our response to the Axios developer tool compromise](https://openai.com/index/axios-developer-tool-compromise/)*

---

我们近期发现了一项涉及第三方开发者工具 Axios 的安全问题，该问题是更大范围行业事件的一部分。出于谨慎考虑，我们正在采取措施保护用于认证 macOS 应用为合法 OpenAI 应用的流程。我们未发现 OpenAI 用户数据被访问、我们的系统或知识产权被入侵、或我们的软件被篡改的证据。

我们正在更新安全证书，这将要求所有 macOS 用户将其 OpenAI 应用更新到最新版本。这有助于防止任何风险——无论多么不可能——有人试图分发伪装成来自 OpenAI 的虚假应用。您可以通过应用内更新或以下官方链接安全更新：

安全和隐私信息是我们的首要任务。我们承诺在问题出现时保持透明并迅速采取行动。我们在下方分享更多技术细节和常见问题。

## 技术详情

3 月 31 日（UTC），广泛使用的第三方开发者库 Axios 作为更广泛的供应链攻击的一部分被入侵。当时，我们用于 macOS 应用签名流程的一个 GitHub Actions 工作流下载并执行了 Axios 的恶意版本（1.14.1）。该工作流可以访问用于签署 macOS 应用的证书和公证材料，包括 ChatGPT Desktop、Codex、Codex CLI 和 Atlas。该证书帮助客户确认软件来自合法开发者 OpenAI。

我们的分析得出结论，由于恶意负载的执行时机、证书注入到作业的时序、作业本身的排序以及其他缓解因素，签名证书很可能未被恶意负载成功窃取。尽管如此，出于谨慎考虑，我们将该证书视为已泄露，并正在撤销和轮换它。

## 受影响产品及最低版本要求

自 2026 年 5 月 8 日起，旧版本的 macOS 桌面应用将不再接收更新或支持，可能无法正常运行。以下为使用更新证书签名的最早版本：

- **ChatGPT Desktop**: 1.2026.051
- **Codex App**: 26.406.40811
- **Codex CLI**: 0.119.0
- **Atlas**: 1.2026.84.2

## 应对措施

作为调查和响应的一部分，我们：

1. 聘请了第三方数字取证和事件响应公司
2. 轮换了 macOS 代码签名证书
3. 发布了所有相关 macOS 产品使用新证书的新版本
4. 与 Apple 合作确保使用旧证书的软件无法获得新的公证
5. 审查了使用旧证书的所有软件公证记录，确认没有意外的软件公证
6. 验证了已发布的软件没有未经授权的修改

目前，我们未发现现有软件安装存在入侵或风险的证据。

## 风险评估

如果签名证书确实被恶意行为者成功窃取，他们可以使用它签署自己的代码，使其看起来像合法的 OpenAI 软件。我们已停止使用旧证书的新软件公证，因此未经授权的第三方使用旧证书签署的新软件将默认被 macOS 安全保护阻止，除非用户明确绕过这些保护。一旦我们在 5 月 8 日完全撤销证书，使用旧证书的新下载和应用启动将被 macOS 安全保护阻止。

## 根本原因

此事件的根本原因是 GitHub Actions 工作流中的配置错误，我们已经解决了该问题。具体来说，相关操作使用了浮动标签，而非特定的 commit hash，并且没有为新包配置 minimumReleaseAge。

## 常见问题

**OpenAI 产品或用户数据是否被入侵？**

没有。我们未发现 OpenAI 产品或用户数据被入侵或暴露的证据。

**是否发现了伪装为 OpenAI 的恶意软件？**

没有。我们未发现可能暴露的公证和代码签名材料被滥用的证据，我们确认使用受影响材料的所有公证事件均为预期事件。

**我需要更改密码吗？**

不需要。密码和 OpenAI API 密钥未受影响。

**这是否影响 iOS、Android、Linux 或 Windows？**

不影响。这仅影响 OpenAI macOS 应用。不影响网页版软件。

**为什么要求更新 Mac 应用？**

OpenAI 发现 macOS 应用签名流程中的 GitHub Actions 工作流存在暴露。由于暴露的工作流与 macOS 应用签名相关，我们正在主动轮换用于 OpenAI macOS 应用的公证和代码签名材料。更新确保您运行的是使用最新证书签署的版本。

**去哪里下载更新的 macOS 应用？**

仅通过应用内更新或以下官方网页下载 OpenAI 应用：

不要从电子邮件、消息、广告或第三方下载站中的链接安装应用。对通过电子邮件、短信、聊天消息、广告、文件共享链接或第三方下载站发送的意外"OpenAI"、"ChatGPT"或"Codex"安装程序保持警惕。

**2026 年 5 月 8 日之后会发生什么？**

自 2026 年 5 月 8 日起，旧版本的 macOS 桌面应用将不再接收更新或支持，可能无法正常运行。

**为什么不立即撤销证书？**

我们已阻止使用受影响公证材料对 macOS 应用进行任何进一步的公证。这意味着使用受影响证书的任何虚假 OpenAI 应用将缺乏公证，因此将默认被 macOS 安全保护阻止。

由于旧证书的新公证已被阻止，并且撤销可能导致 macOS 阻止使用旧证书的新下载和首次启动，我们给用户 30 天的窗口期来更新，以尽量减少中断。此窗口有助于降低用户风险，并允许受影响的客户端通过内置更新机制进行更新。

我们正在与合作伙伴合作监控签名证书被滥用的任何指标，如果在此窗口期间发现恶意活动，我们将加快撤销时间表。
