# OpenAI models, Codex, and Managed Agents come to AWS（中文翻译）

> 原文：<https://openai.com/index/openai-on-aws/>
>
> 说明：本文为 AI 辅助翻译，尽量完整保留原文信息、结构与术语。

今天，OpenAI 与 AWS 正在扩大战略合作，帮助企业在其 AWS 环境中构建基于 OpenAI 能力的系统。我们很高兴为 AWS 客户提供对最佳前沿模型、agents 和工具的访问，这些能力将运行在他们已经在使用的系统、安全协议、合规要求和工作流之中。

此次与 Amazon 扩大的合作将三个关键方向结合在一起，且都于今天以 limited preview 形式推出：

- OpenAI models on AWS
- Codex on AWS
- Amazon Bedrock Managed Agents, powered by OpenAI

这些能力合在一起，让组织能够在应用开发、软件工程和 agentic workflows 中，以更多方式使用 OpenAI，同时仍然基于他们已经在 AWS 中使用的基础设施、安全、治理和采购流程进行构建。

对许多公司来说，要大规模使用 AI，意味着必须把最好的模型带入团队已经在使用的系统中。这就是为什么我们正在 Amazon Bedrock 上推出 OpenAI 模型，包括我们最好的前沿模型 GPT-5.5。

客户现在可以在 AWS 中使用 OpenAI 模型构建产品，并与他们已经依赖的服务、安全控制、身份系统和采购流程协同使用。

对开发者而言，这意味着他们在使用 OpenAI 时拥有更大的构建灵活性，无论是构建新的 AI 应用、把智能嵌入现有产品，还是构建能够推理、采取行动并支持更复杂业务流程的 agentic workflows。

对企业而言，这意味着从实验走向生产将拥有一条清晰而统一的路径，因为 OpenAI 的能力可以直接出现在 AWS 环境中，而这些环境正是其最重要工作负载已经运行的地方。

如今每周已有超过 400 万人使用 Codex，各团队正在把它应用于软件开发生命周期的多个环节，包括编写代码、解释系统、重构应用、生成测试、现代化遗留代码库，以及加速一系列超出编码本身的专业工作流。越来越多的人也在使用 Codex 来加速研究、分析和文档相关工作，通过连接他们日常使用的应用和工具，从汇总源材料到制作 briefing、演示文稿和电子表格。

Codex 是 OpenAI 的前沿 coding harness 和产品套件，而现在组织可以让 Codex 直接使用由 Amazon Bedrock 提供的 OpenAI 模型。这使得任何拥有 AWS commit 和 Bedrock 访问权限的公司，都可以几乎无摩擦地开始使用 OpenAI 强大的 coding agent 和相关产品。

客户只需将 Codex 配置为以 Bedrock 作为 provider 即可开始使用。这样，客户就能获得他们期望 AWS 提供的企业级特性，包括安全、计费和高可用性。所有客户数据都由 Amazon Bedrock 处理，符合条件的客户还可以将 Codex 使用量计入其 AWS cloud commitments。

Codex on Bedrock 目前处于 limited preview。客户可通过 Bedrock API 将 Codex 配置为使用 Amazon Bedrock，起始支持包括 Codex CLI、Codex 桌面应用和 Visual Studio Code 扩展。

我们也在推出 Amazon Bedrock Managed Agents, powered by OpenAI，为企业提供一种新方式，在其可信的 AWS 环境中部署先进 agents。

借助 Bedrock Managed Agents，组织可以构建能够保持上下文、执行多步骤工作流、使用工具，并在复杂业务流程中采取行动的 agents。这帮助客户更快地从实验走向生产，同时确保 agent 开发与他们对 AWS 所期待的基础设施、安全和运营标准保持一致。

对企业来说，Bedrock Managed Agents 让团队能够把重点放在让 agents 真正服务于实际工作上，而不是自己去搭建周边基础设施。它处理了部署、工具使用、编排和治理中更困难的部分，并内建集成了 Amazon 的安全与合规控制。因此，企业可以更快地把能在真实环境中运行的 agents 从原型推进到生产。

我们与 Amazon 的战略合作聚焦于帮助组织以生产级规模部署先进 AI。随着 OpenAI 模型、Codex 和 Managed Agents 进入 Amazon Bedrock，客户将拥有一条更快、更安全的路径，把 AI 真正投入业务运作中。我们很期待看到你们的组织会构建出什么。
