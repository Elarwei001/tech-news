# Codex for (almost) everything（中文翻译）

> 原文：<https://openai.com/index/codex-for-almost-everything/>

---

我们正在发布一次针对 Codex 的重大更新，让它成为一个更强大的合作伙伴，服务于每周使用它来加速整个软件开发生命周期工作的 300 多万开发者。

Codex 现在可以在你身边操作你的电脑，能够使用你日常会用到的更多工具和应用，生成图片，记住你的偏好，从之前的操作中学习，并承担持续性和可重复的工作。Codex 应用现在也对开发者工作流提供了更深入的支持，例如审查 PR、查看多个文件与终端、通过 SSH 连接远程 devbox，以及内置浏览器，从而更快地迭代前端设计、应用和游戏。

借助后台计算机操作能力，Codex 现在可以通过自己的光标“看见”、点击和输入，从而使用你电脑上的所有应用。多个 agent 可以并行在你的 Mac 上工作，同时不会干扰你在其他应用中的工作。对于开发者来说，这有助于迭代前端改动、测试应用，或在那些没有暴露 API 的应用里完成工作。

Codex 也开始原生支持 Web。应用现在包含一个内置浏览器，你可以直接在页面上评论，从而给 agent 提供更精确的指令。这对于前端和游戏开发已经很有帮助，未来我们还计划继续扩展，让 Codex 不只是操作 localhost 上的 Web 应用，而是能更完整地掌控浏览器。

Codex 现在可以使用 [gpt-image-1.5](https://developers.openai.com/api/docs/models/gpt-image-1.5) 来生成和迭代图片。结合截图和代码，它可以在同一工作流中帮助创建产品概念图、前端设计、mockup 和游戏视觉素材。

我们还发布了 90 多个额外插件，这些插件将 skills、应用集成和 MCP servers 结合起来，让 Codex 有更多方式去获取上下文，并在你的工具体系中采取行动。开发者会觉得特别有用的一些新插件包括 Atlassian Rovo（帮助管理 JIRA）、CircleCI、CodeRabbit、GitLab Issues、Microsoft Suite、Neon by Databricks、Remotion、Render 和 Superpowers。

应用现在还在 alpha 阶段加入了更多开发者能力，包括处理 GitHub review comments、运行多个终端标签页，以及通过 SSH 连接远程 devbox。它还支持在侧边栏直接打开文件，并对 PDF、电子表格、幻灯片和文档提供丰富预览，同时新增了 summary pane，用于跟踪 agent 的计划、来源和产物。

这些改进结合在一起，使开发者能在软件开发生命周期的各个阶段之间更快切换, 从写代码、检查结果、审查变更，到与 agent 在同一个工作空间中协作。

我们已经扩展了 automations，允许复用现有会话线程，从而保留之前已经建立的上下文。Codex 现在可以为自己安排未来的工作，并在之后自动唤醒，继续一个长期任务，这个周期甚至可能跨越数天或数周。

团队正在把 automations 用在各种场景里，从合并未完成的 pull request，到跟进任务，以及处理 Slack、Gmail 和 Notion 等工具中快速变化的对话。

我们还发布了 memory 的预览版，它允许 Codex 记住以前经验中有用的上下文，包括个人偏好、修正意见，以及那些原本需要花时间收集的信息。这有助于未来的任务更快完成，并达到过去只有通过大量自定义指令才能实现的质量水平。

Codex 现在还会主动提出有用的后续工作，帮助你从之前中断的地方继续。利用来自项目、已连接插件和 memory 的上下文，Codex 现在可以建议你如何开始一天的工作，或从哪里继续一个之前的项目。例如，Codex 可以识别 Google Docs 中需要你处理的未解决评论，从 Slack、Notion 和你的代码库中提取相关上下文，然后给你一份按优先级排序的行动列表。

从今天开始，这些更新将逐步向已登录 ChatGPT 的 Codex 桌面应用用户开放。

包括基于上下文的建议和 memory 在内的个性化功能，也将很快向 Enterprise、Edu，以及欧盟和英国用户开放。计算机操作能力目前首先在 macOS 上提供，也会很快向欧盟和英国用户开放。

如果你一直在终端或编辑器中使用 Codex，不妨试着把它带到你工作流的其他部分。如果你还没试过 Codex，可以[下载应用](https://openai.com/codex/)并开始使用。

就在 Codex 发布后的这一年里，开发者使用 Codex 的方式已经扩展了很多。开发者最初使用 Codex 是为了写代码，而现在，他们越来越多地用它来理解系统、收集上下文、审查工作、调试问题、与队友协同，以及推动长期运行的工作持续前进。

我们的使命是确保 AGI 惠及全人类。这也包括缩小“人们能想象什么”和“人们能构建什么”之间的差距。这次发布让 Codex 更接近软件构建过程中所涉及的工具、工作流和决策，而未来还会有更多更新很快到来。
