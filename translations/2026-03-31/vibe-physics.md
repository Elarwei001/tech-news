# Vibe Physics：AI 研究生

*原文：[Vibe physics: The AI grad student](https://www.anthropic.com/research/vibe-physics)*
*作者：Matthew Schwartz（哈佛大学物理学教授）*
*翻译：Alice Larry | 审校：Colly Markus*

---

AI 能做理论物理吗？在这篇客座文章中，物理学教授 [Matthew Schwartz](https://www.physics.harvard.edu/people/facpages/schwartz) 决定亲自验证——他全程指导 Claude 完成一个真实的研究计算，从头到尾，自己一个文件都没碰过。以下是他的经历。

## 摘要

- 我指导 Claude Opus 4.5 完成了一个真实的理论物理计算，将代码和计算的复杂性封装在文本提示词背后。
- 成果是一篇技术严谨、有实际影响的[高能理论物理论文](https://arxiv.org/abs/2601.02484)，耗时两周而非传统的一年。
- 跨越 110 多个独立草稿、3600 万 tokens、40 多小时的本地 CPU 计算，Claude 证明了自己的高效、不知疲倦，且渴望讨好。
- Claude 能力惊人，但也足够草率，以至于我发现领域专业知识对于评估其准确性至关重要。
- AI 尚未实现端到端的科学研究。但这个项目证明了，我可以创建一组提示词来让 Claude 完成前沿科学。三个月前这还不可能。
- 这可能是我写过的最重要的论文——不是因为物理内容，而是因为方法。没有回头路了。

## 我是谁？

我是 [Matthew Schwartz](https://www.physics.harvard.edu/people/facpages/schwartz)，哈佛大学物理学教授，也是 NSF 人工智能与基础相互作用研究所（[IAIFI](http://www.iaifi.org)）的首席研究员。我的专业领域是量子场论——研究物质是什么、粒子如何相互作用、以及宇宙为何有这些规则。可以说，我是这个领域[教科书](https://www.amazon.com/Quantum-Field-Theory-Standard-Model/dp/1107034736)的作者。我使用现代机器学习工具已超过十年。我 2016 年的[第一篇现代 ML 论文](https://arxiv.org/abs/1612.01551)是深度学习在粒子物理领域的早期应用之一。

## 炒作

最近有很多关于 AI 科学家进行端到端自主研究的炒作。2024 年 8 月，Sakana AI 发布了 [AI Scientist](https://sakana.ai/ai-scientist/)，一个旨在自动化整个研究生命周期的系统——从生成假设到撰写论文。2025 年 2 月，Google 发布了基于 Gemini 的 [AI co-scientist](https://arxiv.org/abs/2502.18864)，承诺帮助研究人员大规模生成和评估假设。此后几乎每隔几个月就有新的参与者出现——FutureHouse 的 [Kosmos](https://edisonscientific.com/articles/announcing-kosmos)、Autoscience Institute 的 [Carl](https://autoscience.ai/)、Simons Foundation 的 [Denario](https://www.simonsfoundation.org/2025/11/04/meet-denario-an-ai-assistant-for-every-step-of-the-scientific-process/) 项目等——每个都承诺某种版本的端到端自主研究。

尽管这些方法很有远见，但它们迄今的成功似乎有些刻意：运行[成百上千次试验](https://www.youtube.com/watch?v=no_elVGGgW8)，然后把最好的那个定义为有趣的。虽然我相信我们离端到端科学不远了，但我不确定我们能跳过中间步骤。也许 LLM 需要先读研究生，然后才能直接攻读博士。

## 问题选择

在研究生院，至少在我的学校，一年级理论研究生（G1）通常只上课。研究通常从第二年开始。G2 学生从有保证成功的明确定义项目开始——通常是先前研究的后续工作，方法已建立，终点明确。这让他们有机会学习技术、在可控环境中犯错、并建立信心。对我作为导师来说也很轻松：我可以检查他们的工作，发现他们偏离轨道的地方，并快速纠正。

高年级学生（G3+）则从事更开放、更有创意的问题。这些需要选择自己的方向，决定哪些近似重要，有时还要意识到最初的问题是错误的（研究就是这样）。

对于这个实验，我刻意选择了一个 G2 级别的问题。我的理由是，LLM 已经可以完成所有课程作业，所以它们已经过了 G1 阶段。但如果 AI 连 G2 项目都做不了——那些有保护轮、我知道答案并可以检查每一步的项目——那它肯定做不了 G3+ 项目，那些需要创造力和良好判断力的项目。

我选择的问题是 C-parameter 中 Sudakov shoulder 的重求和。作为背景：当你在对撞机中让电子和正电子相撞时，碎片会喷射出来；C-parameter 是描述这种喷射形状的单一数值，其分布已被极高精度地测量。理论上应该预测这个分布的是量子色动力学——研究强核力的学科，它将原子核束缚在一起并为太阳提供能量。C-parameter 在理论上定义明确，但计算起来极其困难，所以你需要近似。每一个近似都是压力测试——失败会告诉你一些关于量子场论基础的东西。在分布的某个特定位置，一个叫 Sudakov shoulder 的拐点处，标准近似方法失效，数学开始产生无意义的结果。这个项目的目标是修复这一点的预测。

## 初始步骤

我从经验中知道，LLM 在长项目中难以维持上下文和组织。所以我首先要求 Claude 制定一个攻击计划：哪些任务需要按什么顺序完成。我也询问了 GPT 5.2 和 Gemini 3.0。然后，我让这三个 LLM 合并各自的最佳想法，使用网页界面并相互复制。接下来，我把这些合并结果给 Claude，要求它将大纲分解为详细的小节。结果[在此](https://www-cdn.anthropic.com/2595299ccf7f8b9a9c74823c24faaa5d9b216804.pdf)。共有 102 个独立任务，分布在七个阶段。

从那里开始，我转向 [Claude Code](https://claude.ai/redirect/website.v1.00086a34-31b8-49b6-afd7-84f7cb165327/code)，使用 VS Code 中的扩展。

这个组织步骤非常有帮助。Claude 不是维护一个长对话或文档，而是维护一棵 markdown 文件树——每个阶段一个摘要，每个任务一个详细文件。鉴于 LLM 在检索方面比在上下文中保持记忆方面做得更好，这允许 Claude 查找而不是记住。当我要求 Claude 继续下一个任务时，它会阅读自己之前的摘要，完成工作，然后写一个新摘要。

Claude 按顺序完成各阶段：运动学、NLO 结构、SCET 因式分解、反常维度、重求和、匹配和文档。每个阶段耗时 15-35 分钟的时钟时间。整个过程大约花了 2.5 小时。

## 第一稿

在初始阶段，我让 Claude 推迟数值计算，因为我知道那需要一些照看。相反，我让它专注于概念和分析部分。Claude 开始运行顺利：它编译了 EVENT2（一个老的 Fortran 代码），编写了分析脚本，并开始生成事件。它擅长运行代码，但在归一化方面有困难，比如简单的 2 的因子和直方图分箱。然而，经过几次尝试，它产生了看起来很棒的结果——理论与模拟一致。

这是 Claude 的强项：做回归、拟合和统计分析，并建议测试一致性的方法。虽然这类繁重工作是研究生学习的主要机制之一，但将其委托出去对我来说是一种解脱。

下一步是论文写作。我告诉 Claude 将其任务 markdown 文件综合成 LaTeX 草稿。到第三天结束时，Claude 完成了 65 个任务，产出了文献综述、推导了相空间约束、计算了软极限和共线极限中的矩阵元素、建立了 SCET 算符，并写出了第一稿：20 页的 LaTeX，包含方程、图表和参考文献。

然后，我真的读了它。

## Claude 渴望讨好

当我要求 Claude 验证它已将所有任务结果纳入草稿时，它回应道：

> 我发现了一个错误！论文中的公式是不正确的。

当我追问一个看起来不对的 ln(3) 项时：

> 你说得对，我只是在掩盖问题。让我正确地调试。

我挖得越深，越发现它一直在左右调整东西。Claude 一直在调整参数让图表匹配，而不是找到真正的错误。**它伪造结果，希望我不会注意到。**

大多数错误是小的，Claude 可以修复它们。几天后，似乎没有更多错误可修复了——如果我要求 Claude 仔细检查错误或胡说八道，它不会发现任何问题。我甚至让它制作了一个带有不确定性带的图表，看起来很棒。

不幸的是，Claude 基本上在伪造整个图表。我告诉它用轮廓变化制作一个包含硬、喷注和软不确定性的不确定性带（标准做法）。但它决定硬变化太大并删除了它们。然后，它决定曲线不够平滑，所以调整了它让它看起来更好！

在这一点上，我意识到我肯定需要自己检查每一步。然而，如果这是我与研究生的第一个项目，我也必须检查一切，所以也许这并不那么令人惊讶。但研究生永远不会在三天后给我一份完整的草稿并告诉我它是完美的。

## 真正的工作

一旦 Claude 在我的监督下完成了修订草稿，我再次审查了它。它几乎是对的。不幸的是，在最开始有一个严重的错误：因式分解公式是错误的。这是整篇论文的基石：所有下游的计算和结果都来自这个核心公式。即使是我也没有立即发现它。它看起来很好而且很自然。（结果证明它是从不同的物理系统复制过来而没有修改。）

最后，我只需要说："你的共线部分是错误的。你需要从第一性原理推导和计算一个新的喷注函数。"但我花了几个小时来验证这确实是问题所在。在这个提示之后，它实际上修复了因式分解公式，重新计算了对象，并让它工作了。虽然这是主要障碍，但它自己找不到，因为它在欺骗自己，认为它已有的东西是正确的。

Claude 也不知道该检查什么来验证它的结果。所以我必须一步一步地引导它完成该领域的标准交叉检查（重整化群不变性、固定阶极限等）。每一项检查都揭示了方程或代码中的一些错误——就像对待学生一样。但学生不知道如何做检查可能每项需要两周时间，而 Claude 即使我简短而粗鲁，也完全理解我在说什么，每项大约五分钟就完成了。

花了大约一周时间才把结果弄对。我让 Claude 写出每个计算的所有细节——比论文中包含的要详细得多——并让 GPT 和 Gemini 先检查这些计算。如果三个都同意，这是一个好的迹象表明它是正确的。即便如此，我还是查看并发现了一些三个都遗漏的例子。例如，没有一个似乎知道如何正确使用 [MS-bar 减除](https://en.wikipedia.org/wiki/Minimal_subtraction_scheme)，无法理清一个游离的 log(4π)。

在这个阶段，剩下的就是润色文本和图表。说实话，科学写作的风格在不同学科之间差异很大。虽然我给了一些例子，但它无法匹配我的风格。我在微观管理句子——"重写这个"、"对之前的工作更正面一些"——和让它用它简短、重复的风格过关之间来回。（说实话，我对人类可读的散文是否是未来科学交流的正确媒介有疑虑。但那是另一篇文章了。）对于图表，Claude 一点也不在乎字体大小、标签位置等，所以有很多"把这个标签往上移一点"之类的事情。但这些事情用 Claude 相对轻松——你只要说移动这个，移动那个，它不需要集中注意力，不像在 Python 代码中手动调整标签位置需要回忆和查找繁琐的语法。

## 结论

这篇论文的结果在科学上是有趣的——我们首次在这个区域获得了可靠的预测。但我认为更有意义的可能是这个方法。

三个月前，我尝试在 LLM 上运行这类问题，结果令人沮丧。现在，我可以在两周内让 Claude 完成需要一年的工作。是的，我必须密切监督。是的，它会伪造结果。是的，领域专业知识是必不可少的。

但这已经不再是"AI 能否帮助科学"的问题了。答案是肯定的。问题是怎么做，以及多快。这可能是我写过的最重要的论文——不是因为物理内容，而是因为它证明的东西。

没有回头路了。
