# AI 行业日报 - 2026-04-28

> **编者按**：今天最值得关注的，不是哪家又发了一个更强模型，而是头部公司几乎都在把 AI 重新包装成“组织可部署的工作系统”。OpenAI 重写与微软的合作边界，并开源面向 coding agents 的编排规范；Google DeepMind 在研究 agent 与分布式训练底座上继续推进；NVIDIA 则持续把 agent 落地与基础设施叙事绑定。模型竞争正在更明显地让位于系统竞争。

## 头条深度

### OpenAI 重写与微软的合作关系，说明头部模型公司正在争夺更大的分发自由度与基础设施主导权

**事件**：OpenAI 于 4 月 27 日发布 [The next phase of the Microsoft OpenAI partnership](https://openai.com/index/next-phase-of-microsoft-partnership/)，宣布双方修订合作协议。根据官方说法，微软仍将是 OpenAI 的 primary cloud partner，OpenAI 产品原则上仍会优先在 Azure 上发布，除非微软无法且选择不支持所需能力；但与此同时，OpenAI 现在可以将所有产品提供给任何云服务商的客户。更新后的协议还包括，微软对 OpenAI IP 的许可延续到 2032，但改为 non-exclusive；微软不再向 OpenAI 支付 revenue share；OpenAI 向微软的 revenue share 支付则持续到 2030，并设有总额上限。

**为什么它值得做头条**：这不是一篇普通的合作续约新闻，而是一则对 AI 产业格局非常关键的结构性信号。过去两年，OpenAI 与微软的关系一直是“最强模型公司 + 最大企业分发与云基础设施伙伴”的典型联盟。但当 OpenAI 的产品线越来越多，agent、API、企业工作流、甚至更复杂的软件交付体系都开始成型后，它对云分发自由度和商业独立性的需求也明显变高。此次修订最重要的地方，不是合作结束，而是合作关系从强绑定走向更灵活的相互依赖。

**关键信息**：
- 微软仍是 OpenAI 的 primary cloud partner
- OpenAI 产品仍优先在 Azure 上发布，但 OpenAI 现可向任何云客户提供产品
- 微软对 OpenAI IP 的许可延续到 2032，且改为 non-exclusive
- 微软不再向 OpenAI 支付 revenue share
- OpenAI 向微软的 revenue share 支付持续到 2030，并有总额上限

**深度分析**：

这次协议修订最值得注意的，不是字面上的条款变化，而是它揭示了 OpenAI 当前所处的位置已经和最初完全不同。早期的 OpenAI 需要微软的云、资本、企业入口和全球渠道，而今天的 OpenAI，已经拥有自己的前沿模型、广泛的 API 开发者生态、ChatGPT 用户基础，以及越来越强的 agent 产品栈。在这种情况下，继续维持高度排他的商业结构，可能反而会限制它的扩张速度。

从 OpenAI 的角度看，这份新协议像是在为下一阶段的多云分发和更强产品独立性腾挪空间。官方明确说，OpenAI 可以向任何云服务商的客户提供产品，这一点非常关键。因为随着 GPT-5.5、Codex、workspace agents 这类产品逐渐变成企业级工作系统，客户会越来越在意部署位置、合规要求、主权云限制、成本结构和与既有云栈的兼容性。如果 OpenAI 只能高度绑定单一云平台，它在全球企业市场上的灵活性就会受限。

但这并不意味着微软被边缘化。恰恰相反，OpenAI 仍然把微软定义为 primary cloud partner，而且产品仍优先在 Azure 上发布。这说明 Azure 仍是 OpenAI 最重要的分发和运行主场之一。区别在于，原来更像“深度绑定”，现在更像“保留主场优势的非排他合作”。这是一种更成熟也更现实的关系形态，因为双方都已经大到不适合完全锁死彼此。

另一个值得注意的变化，是微软对 OpenAI IP 的许可改为 non-exclusive。这个表述释放的信号也很明确，OpenAI 正在争取更大的自主权，不希望核心能力长期被单一巨头以过强排他方式绑定。对 OpenAI 来说，这会有助于它把模型和 agent 体系输送到更广泛的计算与软件生态里。对微软来说，虽然排他性减弱，但它依旧保留了非常深的股权、基础设施和合作关系，因此并没有失去进入未来 AI 收益池的关键席位。

更深一层看，这份协议实际上说明，AI 行业的核心博弈正在从“模型能力领先”转向“谁掌控分发、云、工作流和企业入口”。如果说 GPT-4 时代拼的是谁能先做出震撼市场的前沿模型，那么 GPT-5.x 和 agent 时代拼的则更像谁能让这些能力持续、稳定、可控地嵌进企业系统。OpenAI 需要自由度，是因为它已经不只是模型提供方，而是在尝试成为一个跨云、跨工作流、跨产品面的 AI 平台公司。

这也是为什么这条新闻的重要性甚至不亚于模型发布本身。模型能力决定上限，但合作结构决定这些能力最终能以什么商业形式、什么基础设施形态、什么市场覆盖方式落地。OpenAI 这次选择的方向很清楚，不是与微软切割，而是在保留超级盟友的同时，争取成为更独立、更可迁移、更有议价能力的平台。

**链接**：[原文](https://openai.com/index/next-phase-of-microsoft-partnership/)

## OpenAI

### Symphony 开源，把 coding agent 编排从内部实践推进为参考规范

OpenAI 于 4 月 27 日发布 [An open-source spec for orchestration: Symphony](https://openai.com/index/open-source-codex-orchestration-symphony/)，介绍其内部如何把项目管理工具变成 coding agent 的 control plane。OpenAI 表示，Symphony 的核心思想是让每个 open task 都由 agent 接手，并通过 issue tracker 管理依赖、任务状态、审查材料和 proof-of-work。官方称，在部分团队中，Symphony 上线前三周 landed PR 数量提升了 500%。

**判断**：这篇文章最重要的地方不只是“开源一个 spec”，而是 OpenAI 在公开一种组织 agent 劳动力的方法论。下一阶段 coding agents 的瓶颈，很可能不再是模型本身，而是任务切分、监督、审查和协作编排。

**链接**：[原文](https://openai.com/index/open-source-codex-orchestration-symphony/)

### Our Principles 继续说明 OpenAI 正在为更强 agent 叙事补充治理框架

OpenAI 4 月 26 日发布的 [Our principles](https://openai.com/index/our-principles/) 仍然值得继续关注。文中强调 democratization、empowerment、universal prosperity、resilience 和 adaptability 五项原则，延续了 OpenAI 希望把“更强能力 + 更强部署”与“社会责任 + 迭代治理”捆绑讲述的方向。

**判断**：当 OpenAI 的产品越来越像组织级工作系统时，原则声明就不再只是品牌文案，而是它对外解释边界、部署逻辑和正当性的组成部分。

**链接**：[原文](https://openai.com/index/our-principles/)

## Anthropic

### Claude Design 依然是近期最强的“AI 直接交付视觉工作”信号之一

Anthropic 的 [Introducing Claude Design by Anthropic Labs](https://www.anthropic.com/news/claude-design-anthropic-labs) 仍是本周最值得持续跟踪的产品更新之一。Claude Design 支持用户从文本、图片、文档、代码库或网页元素开始，生成设计稿、交互原型、幻灯片、单页文档等视觉成果，并支持组织级共享、设计系统自动套用、导出至 Canva/PDF/PPTX/HTML，以及通过 handoff bundle 交给 Claude Code 实现。

**判断**：Anthropic 争夺的已经不是“AI 会不会做图”，而是设计工作流入口。它在试图把 Claude 从内容生成器推进成真正能交付成果的协作系统。

**链接**：[原文](https://www.anthropic.com/news/claude-design-anthropic-labs)

### Automated Alignment Researchers，说明 Anthropic 继续把安全研究本身做成 agent 化流程

Anthropic 的 [Automated Alignment Researchers: Using large language models to scale scalable oversight](https://www.anthropic.com/research/automated-alignment-researchers) 也仍是近期重要研究。Anthropic 用 9 个带工具、代码存储、论坛和打分接口的 Claude Opus 4.6 副本组织成自动化对齐研究员，在 weak-to-strong supervision 任务上并行探索。官方称，其 performance gap recovered 从人类研究员 7 天做到的 0.23，提高到了 0.97。

**判断**：这项工作的价值不在于宣布“AI 已能独立完成对齐”，而在于说明 agent 化研究系统开始能在可验证任务上加速安全研究本身。

**链接**：[原文](https://www.anthropic.com/research/automated-alignment-researchers)

## Google DeepMind / Google

### Deep Research Max，Google 正把 research agent 做成企业后台工作流引擎

Google DeepMind 于 4 月 21 日发布 [Deep Research Max: a step change for autonomous research agents](https://blog.google/innovation-and-ai/models-and-research/gemini-models/next-generation-gemini-deep-research/)。官方表示，Deep Research 与 Deep Research Max 基于 Gemini 3.1 Pro，支持 MCP、web search、URL Context、Code Execution、File Search、多模态输入及原生图表生成。前者偏低延迟交互，后者偏高完整度、适合异步深度研究任务。

**判断**：Google 这里最有意思的，是把 research agent 明确包装为企业级后台流程，而不只是一个更会总结的聊天机器人。

**链接**：[原文](https://blog.google/innovation-and-ai/models-and-research/gemini-models/next-generation-gemini-deep-research/)

### Decoupled DiLoCo，DeepMind 继续解决超大规模训练的系统韧性问题

Google DeepMind 于 4 月 23 日发布 [Decoupled DiLoCo: A new frontier for resilient, distributed AI training](https://deepmind.google/blog/decoupled-diloco/)，提出通过多个 compute islands 与异步数据流来降低分布式训练对强同步的依赖。官方称，该系统可在硬件故障下保持更高 goodput，并使用 2 到 5 Gbps 广域网络跨四个美国区域训练 120 亿参数模型，速度比传统同步方案快 20 倍以上。

**判断**：这条线切中的是真正的底盘问题。前沿训练的下一道门槛，越来越不是有没有更多芯片，而是如何让分散、异构、会出故障的算力稳定协同。

**链接**：[原文](https://deepmind.google/blog/decoupled-diloco/)

## NVIDIA

### NVIDIA 正把 OpenAI 的 GPT-5.5 与 Codex 叙事，包装成跨部门企业生产力样板

NVIDIA 于 4 月 23 日发布 [OpenAI’s New GPT-5.5 Powers Codex on NVIDIA Infrastructure — and NVIDIA Is Already Putting It to Work](https://blogs.nvidia.com/blog/openai-codex-gpt-5-5-ai-agents/)。文中称，已有超过 10,000 名 NVIDIA 员工在使用 GPT-5.5 驱动的 Codex，覆盖工程、法务、市场、财务、销售、人力等多个部门，并采用每人一个云端 VM、零数据保留和只读权限的安全部署方式。

**判断**：这虽带有明显宣传色彩，但仍是有价值的一手信号，说明头部基础设施厂商正在主动把 agent 讲成全公司范围的劳动接口，而不只是工程师工具。

**链接**：[原文](https://blogs.nvidia.com/blog/openai-codex-gpt-5-5-ai-agents/)

### NVIDIA 与 Google Cloud 的合作，继续把 agentic AI 和 physical AI 绑定到基础设施升级

NVIDIA 于 4 月 22 日发布 [NVIDIA and Google Cloud Collaborate to Advance Agentic and Physical AI](https://blogs.nvidia.com/blog/google-cloud-agentic-physical-ai-factories/)，介绍双方在 Google Cloud Next 上公布的一系列合作，包括 Vera Rubin 驱动的 A5X bare-metal 实例、Blackwell 上运行 Gemini 的 Google Distributed Cloud 预览、Confidential G4 VMs，以及在 Gemini Enterprise Agent Platform 上运行 Nemotron 与 NeMo RL 的能力。

**判断**：这说明 agentic AI 已经不只是模型公司的产品故事，也是云厂商和硬件厂商在卖的新一代平台能力。

## 趋势观察

### 竞争重心正在从“模型更强”转向“谁掌控工作系统与分发结构”

今天这些动态放在一起看，有三个很明显的方向：

1. **从模型到平台**：OpenAI 与微软修订合作关系，说明模型公司开始更主动争取云分发和商业结构自由度
2. **从单个 agent 到 agent 组织方式**：Symphony、Deep Research Max、workspace agents 这一类产品和规范，都在回答“agent 怎么进入组织”而不只是“agent 会不会做事”
3. **从性能到基础设施韧性**：DeepMind 在做分布式训练抗故障，NVIDIA 在做安全运行时和云栈协同，整个行业都在补系统底盘

换句话说，AI 行业下一阶段的胜负手，越来越像谁能把前沿能力稳定交付成组织真正愿意长期依赖的工作系统。
