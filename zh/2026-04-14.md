## 头条深度

### Anthropic 发布《Trustworthy agents in practice》, 代理安全开始从原则走向产品设计

**事件**：Anthropic 于 4 月 9 日发布研究文章 [Trustworthy agents in practice](https://www.anthropic.com/research/trustworthy-agents)，系统解释其如何把“可信代理”原则落实到实际产品设计中。文章围绕四个代理组成层展开, 模型、harness、工具、环境，并重点讨论三个核心议题, 人类控制、目标理解和提示注入防御。

**背景**：过去一年，行业对 agent 的讨论大多集中在能力跃升，比如能不能写代码、调工具、跨应用执行任务。但 Anthropic 这篇文章的重点不是展示能力边界，而是回答一个更现实的问题, 当代理开始读邮件、动文件、跨系统操作时，应该怎样设计控制面，才能既保留自动化价值，又不把风险外包给用户。

**深度分析**：

这篇文章的重要性，在于它把“代理安全”从口号拉回了工程问题。

第一，Anthropic 明确反对把 agent 风险仅仅理解成模型风险。文章提出，代理行为取决于四层系统一起工作, 模型本身、运行脚手架、可调用工具，以及所处环境。这一点很关键，因为它等于承认，未来 agent 的事故很多未必来自模型“变坏”，而是来自权限配置过宽、环境暴露过多，或产品把确认流程设计得过于形式化。

第二，它开始承认“人类在回路中”这句话本身不够用了。Anthropic 给出的方向是把监督从逐步审批转向计划级审批，例如 Claude Code 的 Plan Mode。这个变化很值得注意。行业早期喜欢把安全理解成多弹几个确认框，但真实世界里，确认框太多只会让人麻木。把人类监督上移到“整体策略”层，反而更接近真正有效的控制。

第三，这篇文章其实是在为 agent 时代的标准化治理打样。Anthropic 点名了三个外部基础设施缺口, 缺少标准 benchmark、缺少证据共享、缺少开放协议层。它再次强调 Model Context Protocol 和开放标准的重要性，这并不只是技术姿态，而是试图把未来 agent 生态的竞争，从“谁垄断集成接口”转向“谁做得更安全、更可靠”。

如果说上周 Project Glasswing 体现的是 Anthropic 对高风险能力的克制发布，那么这篇文章体现的则是另一条更长的路线, 他们想把 agent 变成一套可治理、可审计、可企业部署的系统，而不是只靠模型聪明来硬顶。接下来真正值得观察的，不是 Anthropic 说了什么，而是这些原则能否在更复杂、多代理、长时任务场景里继续成立。

**链接**：[原文](https://www.anthropic.com/research/trustworthy-agents) | [中文翻译](../translations/2026-04-14/anthropic-trustworthy-agents.md) *(AI 译)*

## Anthropic

### Project Glasswing 继续扩张为“防御优先”叙事

Anthropic 官网新闻页当前头条仍是 4 月 7 日发布的 [Project Glasswing](https://www.anthropic.com/glasswing)。这个项目联合 AWS、Apple、Cisco、Google、Microsoft、NVIDIA 等多家机构，目标是用受限访问的 Claude Mythos Preview 去扫描和加固关键软件基础设施。

从今天回看，Glasswing 和《Trustworthy agents in practice》其实构成了 Anthropic 当下非常清晰的双线策略, 一边强调高能力模型必须有限开放、先用于防御，一边强调低一层的 agent 产品必须在权限、计划、环境隔离上做严。这比单纯喊“我们重视安全”更具体，也更像一家准备深度进入企业和政府场景的公司。

### 一个值得注意的信号, Anthropic 正在定义“代理控制面”

文章中特别强调，随着 Claude Code 等产品开始把工作分发给 subagents，未来监督不再只是看一条线性的 action log，而要能理解并干预并行工作流。这是很重要的产品信号, 下一阶段 agent 竞争，可能不只是模型谁更强，而是谁能先做出真正可管理的多代理操作系统。

## OpenAI

### OpenAI 押注企业 AI 的下一阶段, 从单点工具走向统一操作层

OpenAI 于 4 月 8 日发布文章 [The next phase of enterprise AI](https://openai.com/index/next-phase-of-enterprise-ai/)，基本把自己的企业路线说透了。核心判断只有一句话, 企业已经过了实验期，接下来要的不是更多零散 copilot，而是一个统一的 AI intelligence layer。

**关键信息**：
- 企业收入已占 OpenAI 总收入 40% 以上，并预计在 2026 年底前接近与消费者业务持平
- Codex 达到 300 万周活用户
- API 每分钟处理超过 150 亿 token
- OpenAI 认为企业真正需要的是跨系统、跨数据源、持续改进的 agents，而不是碎片化 AI point solutions

**判断**：这篇文章等于是 OpenAI 的企业宣言。它不再只是卖模型 API，也不只是卖 ChatGPT 企业版，而是想成为公司内部所有 agents 的底层 intelligence layer，再向上叠加一个统一 AI superapp。简单说，OpenAI 想卖的不只是“更聪明的模型”，而是“公司的 AI 操作系统”。

### Codex 团队席位改为按量付费，进一步降低企业试点门槛

在 4 月 2 日的 [Codex now offers pay-as-you-go pricing for teams](https://openai.com/index/codex-flexible-pricing-for-teams/) 中，OpenAI 还宣布 ChatGPT Business 和 Enterprise 用户可以为团队添加按 token 消耗计费的 Codex-only 席位，不再需要固定座席费，同时把 ChatGPT Business 年费价格从每席 25 美元下调到 20 美元。

这说明 OpenAI 很清楚一个现实, 企业愿意尝试 agent coding，但预算审批通常不愿先买大包年。按量付费是典型的落地策略优化，也会直接加剧和 Anthropic Claude Code 之间的正面对抗。

## Google DeepMind / Google

### Gemma 3n 全量发布，Google 继续推进端侧多模态 AI

Google DeepMind 相关入口页面今天没有提供稳定、结构化的新文章列表，但通过官方开发者博客可确认，Google 已发布 [Introducing Gemma 3n: The developer guide](https://developers.googleblog.com/en/introducing-gemma-3n-developer-guide/)。

**关键点**：
- Gemma 3n 面向 on-device 场景，支持图像、音频、视频、文本输入
- 提供 E2B 和 E4B 两个 effective parameter 级别，但通过架构优化把显存需求压到更接近传统 2B/4B 模型
- 支持 140 种文本语言，35 种多模态理解语言
- 引入 MatFormer、Per-Layer Embeddings、KV cache sharing、新音频编码器和 MobileNet-V5 视觉编码器

**判断**：Google 这条路线很清晰, 它不只想在云端前沿模型上竞争，也在押注“够强但更轻”的多模态端侧模型。相比 OpenAI 和 Anthropic 更强调 agent 生产力，Google 在 2026 年明显更想拿下 edge AI 和设备级分发入口。

## NVIDIA

### NVIDIA 继续把 agent 与 physical AI 绑进同一套基础设施叙事

NVIDIA 本周有两条值得放在一起看。

第一，官方博客 4 月 10 日发布 [National Robotics Week — Latest Physical AI Research, Breakthroughs and Resources](https://blogs.nvidia.com/blog/national-robotics-week-2026/)，集中展示 Isaac GR00T、Cosmos world models、Newton 1.0、Isaac Sim 6.0、Isaac Lab 3.0 等 physical AI 组件如何联动。

第二，技术博客 4 月 11 日发布 [MiniMax M2.7 Advances Scalable Agentic Workflows on NVIDIA Platforms for Complex AI Applications](https://developer.nvidia.com/blog/minimax-m2-7-advances-scalable-agentic-workflows-on-nvidia-platforms-for-complex-ai-applications/)，强调 MiniMax M2.7 这类 230B MoE 模型如何在 NVIDIA 平台上支撑长上下文 agentic workflows，并与 vLLM、SGLang、NeMo、NIM 等生态打通。

**判断**：NVIDIA 的优势不是自己定义最终 AI 产品，而是持续把“模型层、推理层、仿真层、机器人层”打包成一个越来越难被替代的基础设施栈。今天的关键信号是，agent AI 和 physical AI 在 NVIDIA 这里已经不是两个故事，而是一套共同的算力与工具链故事。

## 趋势观察

### 行业正在从“模型竞争”进入“系统竞争”

今天几家公司的更新拼在一起看，有一个很明显的共同趋势, 大家比的越来越不是单个模型分数，而是谁能把模型变成完整系统。

- Anthropic 在补 agent 的治理与控制面
- OpenAI 在补企业 AI 的统一入口和商业化结构
- Google 在补端侧多模态分发能力
- NVIDIA 在补从 agent 到 robotics 的全栈基础设施

这意味着接下来的竞争维度会更复杂。模型本身当然仍重要，但真正决定企业采购、开发者生态和长期护城河的，越来越会是权限体系、工具协议、部署方式、成本结构，以及谁能把这些拼成一整套顺手的工作流。
