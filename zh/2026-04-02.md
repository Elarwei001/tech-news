## 头条深度

### Claude Code 源代码通过 npm source map 文件意外泄露

**事件**：3 月 31 日，安全研究员 Chaofan Shou 发现 Anthropic 的 Claude Code CLI 工具在 npm 发布包中意外包含了 source map 文件，暴露了完整源代码。

**技术细节**：

泄露源自 `@anthropic-ai/claude-code` npm 包的 2.1.88 版本。一个 59.8 MB 的 JavaScript source map 文件（.map）被意外包含在发布中。Source map 原本用于内部调试，能将压缩后的代码映射回原始源代码——这意味着任何人都可以从 npm 包中还原出完整的原始代码。

**规模有多大**：

- ~1,900 个 TypeScript 文件
- 512,000+ 行代码
- ~40 个内置工具
- ~50 个斜杠命令

泄露的代码迅速被存档到 GitHub 仓库，在数小时内获得 1,100+ star 和 1,900+ fork。社交媒体上相关帖子浏览量超过 2900 万次。

**我们从代码中看到了什么**：

根据开发者社区的分析，Claude Code 的架构远比外界想象的复杂：

1. **工具系统**：约 40 个独立的、基于权限控制的工具（文件读写、Bash 执行、Web 请求、LSP 集成等），单是工具定义就有 29,000 行 TypeScript

2. **查询引擎**：46,000 行代码，是整个代码库中最大的模块，负责所有 LLM API 调用、流式传输、缓存和编排

3. **多 Agent 编排**：Claude Code 可以生成子 Agent（内部称为"swarms"），每个 Agent 在自己的上下文中运行，拥有特定的工具权限

4. **IDE 桥接系统**：通过 JWT 认证的双向通信层，连接 VS Code、JetBrains 等 IDE 扩展

5. **持久化记忆系统**：基于文件的记忆目录，跨会话保存用户、项目和偏好信息

**技术选型亮点**：

- 使用 Bun 而非 Node.js 作为运行时
- 使用 React + Ink 构建终端 UI
- Zod v4 进行全面的 schema 验证
- 重型依赖（OpenTelemetry、gRPC）采用懒加载

**Anthropic 的回应**：

Anthropic 已向 GitHub 发出 DMCA 版权删除请求。但由于泄露代码已被开发者改写为 Python 版本，DMCA 的适用性受到质疑。此外，Anthropic 的删除请求意外波及了大量 fork 仓库，公司表示这是"意外情况"。

**为什么这很重要**：

这起事件的讽刺之处在于：一个帮助开发者写出更好代码的工具，被一个基础的构建配置疏漏击败了。对于每个工程团队来说，这是一个警示——检查你的构建流程，确保 .map 文件被排除在生产包之外。

从竞争角度看，Claude Code 的架构现在已是公开信息。多 Agent 编排、权限系统、IDE 桥接——这些模式对其他 AI 编程工具的开发者来说是宝贵的参考。

不过，代码公开不等于产品可复制。模型能力、基础设施、用户体验的整合才是真正的护城河。

**⚠️ 信息来源说明**：本条新闻基于开发者社区的公开分析。截至发稿时，Anthropic 尚未在官网发布正式声明。

## Anthropic

### Research

**澳大利亚 Claude 使用报告（续）**

昨天我们报道了 Anthropic 发布的澳大利亚经济指数报告。今天补充几个值得关注的细节：

**州际差异揭示的采用模式**：

新南威尔士州（37%）和维多利亚州（31%）占据了近 70% 的 Claude 使用量，但人均使用率与收入并不相关。西澳大利亚州人均 GDP 最高，但 Claude 使用率最低——这反映的是产业结构差异：矿业主导的经济 vs 金融/科技主导的经济。

**"教育水平高但任务时长短"的悖论**：

澳大利亚用户的提示词需要约 11.9 年的教育才能理解（高于全球中位数），但估计的"无 AI 完成时间"只有 2.7 小时（低于全球平均 3.3 小时）。这说明高采用国家的用户更倾向于把 AI 用于"高门槛、快节奏"的任务。

## 趋势观察

**"意外泄露"背后的 AI 工具复杂度**

Claude Code 的代码泄露事件，除了安全警示之外，还揭示了一个事实：现代 AI 编程工具的复杂度已经远超"API wrapper"的范畴。

51 万行代码、40+ 工具、多 Agent 编排、IDE 集成、持久化记忆——这是一个完整的软件平台。当我们在终端里打几个字就能让 AI 修改代码时，背后是这样庞大的工程支撑。

这也解释了为什么 AI 编程工具的门槛越来越高。不是谁都能复制一个 Claude Code——即使代码公开了，模型、基础设施、产品打磨的积累仍然是护城河。
